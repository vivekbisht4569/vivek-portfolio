🎓 University Campus Enterprise Network

This project demonstrates the design and implementation of a real-world university campus network using Cisco Packet Tracer. The infrastructure was built following enterprise networking principles, integrating VLAN segmentation, Router-on-a-Stick (ROAS), centralized DHCP services, Spanning Tree Protocol (STP), and Inter-VLAN Routing. Unlike individual networking labs, this project combines multiple technologies into one scalable campus infrastructure while emphasizing structured troubleshooting and network verification.

Cisco Packet Tracer VLANs ROAS DHCP STP 802.1Q Trunking Inter-VLAN Routing

🎯 Project Objective

The primary objective of this project was to design and deploy a scalable university campus network using enterprise networking concepts. The network was built to provide secure communication between multiple university departments while implementing centralized IP management, Layer 2 loop prevention, and efficient inter-department communication. Rather than focusing only on device configuration, this project emphasized structured troubleshooting and real-world network deployment practices.

The goal of this lab was to implement and understand how Real-World Enterprise Network works .

🏗️ Network Architecture

The university campus network was designed using a hierarchical enterprise architecture consisting of a router, a core switch, two access switches, a centralized DHCP server, and multiple client devices distributed across different university departments.

  • 1 Cisco 2811 Router configured for Router-on-a-Stick (ROAS).
  • 1 Cisco Catalyst 3560 Core Switch acting as the central switching device.
  • 2 Cisco Catalyst 2960 Access Switches providing connectivity to end devices.
  • 1 Centralized DHCP Server responsible for dynamic IP address allocation.
  • 40 End Devices distributed across four different VLANs.
  • 802.1Q Trunk Links connecting the router, core switch, and access switches.
  • Spanning Tree Protocol (STP) implemented for loop prevention.

🧱 VLAN & IP Addressing Plan

To improve security, scalability, and network management, the university departments were logically separated into different VLANs. Each VLAN was assigned an independent subnet and default gateway.

  • VLAN 10 — Administration
    Network: 192.168.10.0/24
    Gateway: 192.168.10.1
  • VLAN 20 — Faculty
    Network: 192.168.20.0/24
    Gateway: 192.168.20.1
  • VLAN 30 — Library
    Network: 192.168.30.0/24
    Gateway: 192.168.30.1
  • VLAN 40 — Computer Lab & DHCP Server
    Network: 192.168.40.0/24
    Gateway: 192.168.40.1

interface fa0/24
 switchport mode trunk

🛠️ Implementation Workflow

  • Designed the complete university campus topology.
  • Created four VLANs representing different university departments.
  • Assigned switch access ports to their respective VLANs.
  • Configured IEEE 802.1Q trunk links between the router, core switch, and access switches.
  • Implemented Router-on-a-Stick (ROAS) for Inter-VLAN Routing.
  • Configured a centralized DHCP server with dedicated DHCP pools for each VLAN.
  • Configured DHCP Relay using ip helper-address for remote VLANs.
  • Enabled Spanning Tree Protocol (STP) to prevent switching loops.
  • Verified end-to-end communication across all departments.

📡 DHCP Deployment

A centralized DHCP server was configured to dynamically assign IP addresses to all VLANs using DHCP pools and IP Helper Address configuration.

🌳 STP & Root Guard

Spanning Tree Protocol (STP) was implemented to prevent Layer 2 loops while Root Guard was configured to protect the network from rogue Root Bridge attacks.
