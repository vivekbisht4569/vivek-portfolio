VLAN Trunking Lab (Multi-Switch)

Overview

This lab demonstrates VLAN segmentation and trunking between two switches using Cisco Packet Tracer. The goal was to allow communication within the same VLAN across switches and restrict communication between different VLANs.

Topology Summary

The network consists of:

  • 2 Switches connected via trunk link (Fa0/24 ↔ Fa0/24)
  • 6 End devices (PCs & Laptops)
  • 3 VLANs: IT (10), HR (20), Sales (30)

VLAN Structure

  • VLAN 10 → IT (192.168.10.x)
  • VLAN 20 → HR (192.168.20.x)
  • VLAN 30 → Sales (192.168.30.x)

What I Configured

1. VLAN Creation

vlan 10
 name IT

vlan 20
 name HR

vlan 30
 name SALES

2. Access Port Assignment

interface fa0/1
 switchport mode access
 switchport access vlan 10

interface fa0/2
 switchport mode access
 switchport access vlan 20

interface fa0/3
 switchport mode access
 switchport access vlan 30

3. Trunk Configuration

interface fa0/24
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30

Testing Results

Successful:

  • Devices in same VLAN communicate across switches
  • Example: VLAN 10 → 192.168.10.2 ↔ 192.168.10.3 ✔

Blocked (Expected):

  • VLAN 10 ✖ VLAN 20
  • VLAN 10 ✖ VLAN 30
  • VLAN 20 ✖ VLAN 30

Key Learning

  • VLANs isolate networks at Layer 2
  • Trunk ports carry multiple VLANs using 802.1Q tagging
  • Access ports belong to only one VLAN
  • Same VLAN devices communicate even across switches
  • Different VLANs cannot communicate without routing

Important Insight

Even after correct VLAN and trunk configuration, devices in different VLANs cannot communicate. This is expected behavior and requires Inter-VLAN Routing using a router or Layer 3 switch.

Where This Is Used

Enterprise networks, campus networks, and data centers where multiple departments need isolation but shared infrastructure.
