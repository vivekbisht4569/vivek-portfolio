IP Subnetting

Designing and dividing networks into smaller subnetworks for better scalability, organization, security, and efficient IP address utilization.

Subnetting CIDR Layer 3 Switching Inter-Subnet Communication

📘 Overview

IP subnetting is the process of dividing a larger network into multiple smaller subnetworks (subnets) to improve efficiency and network management.

Subnetting reduces broadcast traffic, improves scalability, and enables better organization inside enterprise networks.

🧪 Lab Scenario

In this lab, a Class C network was divided into multiple smaller subnets using CIDR notation and configured on a multilayer switch to test communication between subnets.

🌐 Network Division

A /24 network was subnetted into smaller /26 subnetworks.

🛜 Layer 3 Switching

Switch Virtual Interfaces (SVIs) were configured as gateways for inter-subnet communication.

✅ Connectivity Testing

Ping tests verified successful communication between different subnets.

🧠 Subnetting Example

The following network: 

192.168.10.0/24

was divided into 4 smaller subnets using: 

/26

192.168.10.0   - 192.168.10.63
192.168.10.64  - 192.168.10.127
192.168.10.128 - 192.168.10.191
192.168.10.192 - 192.168.10.255

Default Gateway

Configured gateways for communication between networks.

⚙️ Configuration

Layer 3 Switch Configuration

interface gig1/0/1
no switchport
ip address 192.168.10.1 255.255.255.192
no shutdown

Host Configuration

PC1:

IP Address: 192.168.10.10
Subnet Mask: 255.255.255.192
Default Gateway: 192.168.10.1
