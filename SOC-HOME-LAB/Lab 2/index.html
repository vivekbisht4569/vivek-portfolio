🌐 Lab 2 - Network Reconnaissance & Wireshark Traffic Analysis

A practical SOC-style cybersecurity lab focused on network reconnaissance detection, packet analysis, and traffic monitoring using Wireshark, Nmap, Kali Linux, and VirtualBox in an isolated lab environment.

🎯 Objective

The objective of this lab was to simulate attacker reconnaissance activity and analyze generated network traffic using Wireshark. This project helped develop foundational SOC Analyst and Blue Team investigation skills related to packet analysis, TCP/IP communication, reconnaissance detection, and network visibility troubleshooting.

🏗️ Lab Architecture

Kali Linux VM (Attacker)
        │
Nmap / ICMP / TCP Traffic
        │
        ▼
Windows Host Machine
        │
        ▼
Wireshark Packet Analysis

⚙️ Environment Setup

  • Host OS: Windows
  • Attacker Machine: Kali Linux VM
  • Virtualization: VirtualBox
  • Network Mode: Host-Only Adapter
  • Packet Analyzer: Wireshark
  • Recon Tool: Nmap

🔍 Network Reconnaissance Activities

Generated ICMP traffic and reconnaissance scans from Kali Linux to the Windows host machine and analyzed captured traffic inside Wireshark. 

ping 192.168.56.1

nmap 192.168.56.1

🛡️ Detection & Traffic Analysis

  • Captured ICMP Echo Requests and Replies
  • Analyzed TCP SYN packets
  • Observed reconnaissance scanning behavior
  • Detected attacker source IP traffic
  • Filtered packets using Wireshark display filters
  • Analyzed packet details and protocol headers
  • Investigated network visibility issues

📸 Step-by-Step Lab Screenshots

Step 1 — Kali Linux IP Configuration
Verified the Kali Linux attacker machine IP address using the ip a command before starting reconnaissance activities. 
ip a
Step 2 — Windows Host IP Identification
Verified the Windows host machine IP address using ipconfig to confirm communication between attacker and target systems. 
ipconfig
Step 3 — ICMP Packet Capture in Wireshark
Generated ICMP ping traffic from Kali Linux and captured Echo Requests inside Wireshark to analyze packet details and protocol behavior. 
icmp
Step 4 — Nmap Reconnaissance Scan
Performed reconnaissance scanning against the Windows host machine using Nmap and identified active services and open ports. 
nmap 192.168.56.1
Step 5 — TCP SYN Packet Analysis
Captured and analyzed TCP SYN packets generated during reconnaissance activity to understand how scanners probe target systems. 
tcp.flags.syn == 1
Step 6 — SYN Scan Detection in Wireshark
Detected reconnaissance scanning behavior by filtering SYN packets and observing rapid port probing activity from the attacker machine. 
tcp.flags.syn == 1
Step 7 — Attacker Traffic Filtering
Filtered network traffic originating specifically from the Kali Linux attacker machine using source IP-based filtering in Wireshark. 
ip.addr == 192.168.56.101
